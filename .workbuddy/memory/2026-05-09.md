## main.cpp 重构为 Application + Gui 类架构
- 参照 example_src 的分层模式，将 258 行单文件 main.cpp 拆分为 4 个文件：
  - `main/main.cpp`（~10行极简入口）
  - `main/core/Application.h/cpp`（App 生命周期、主循环、菜单栏渲染）
  - `main/ui/Gui.h/cpp`（Win32 窗口、DX11 初始化/清理、ImGui 上下文、消息泵、BeginFrame/EndFrame）
  - `main/GuiTimeConverter.h/cpp`（不变）
- Gui 类封装全部平台细节，DX11 globals 为文件域 static（与 example_src 一致）
- Gui::BeginFrame() 返回 bool，false 表示 WM_QUIT，驱动 Application::Run() 的 while 循环退出
- Application::Render() 负责菜单栏和子窗口渲染，后续新功能在此扩展
- 注意：Gui.h 需 include <windows.h>（HWND/WNDCLASSEXW 类型）；CreateRenderTarget 需前向声明

## 新增 Coord Converter 窗口
- 文件：main/GuiCoordConverter.h/.cpp，与 GuiTimeConverter 同级
- 复用项目 lib/ 中的 CoordConvert.h（XYZtoBLH/BLHtoXYZ/XYZtoENU）和 CoordStruct.h（XYZ/BLH/ENU 类 + Frame::WGS84/GPS/CGCS2000/PZ90）
- 功能：
  - 顶部 Combo 选择参考系（WGS84/GPS/CGCS2000/PZ90），切换自动重新计算
  - XYZ ↔ BLH 双向转换，修改 XYZ 自动算 BLH，或点 BLH->XYZ 按钮反向转换
  - 给定参考点（BLH）和目标点（XYZ），自动计算 ENU 结果（E/N/U 只读）
- 注意：CoordConvert.h 的 PI 依赖 Const.h，需在 include 时先 include Const.h 再 include CoordConvert.h
